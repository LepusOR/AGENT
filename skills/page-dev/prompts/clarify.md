# AskUserQuestion 触发场景清单

命中任一条时停下问用户，不得自动决断。每次停顿必须给出具体上下文（文件路径、行号、候选方案），不使用「是否继续」类空泛问法。

## 通用场景

1. **需求类型不清**
   - 例：无法判断 bug / feat / 重构
   - 选项：按 bug 处理 / 按 feat 处理 / 用户补充
2. **目标页面不清**
   - 例：页面名未匹配 `references/pages-map.md`，或一个 PRD 匹配多个页面
   - 选项：列出候选页面 / 用户输入页面名 / 取消
3. **跨页面范围过大**
   - 例：需要跨 2 个以上页面改动
   - 选项：继续并分批 / 拆分任务 / 取消
4. **业务逻辑风险**
   - 例：改动可能影响权限、接口语义、数据兼容、Form 值结构
   - 选项：保守实现 / 按 PRD 扩展 / 用户指定策略
5. **规则与既有代码冲突**
   - 例：既有文件大量 default export，本次是否顺带改 named export
   - 选项：仅改本次涉及部分 / 顺带修复整文件 / 保留现状
6. **自检无法自动修复**
   - 例：回修 2 轮仍失败，或自动修复可能破坏业务逻辑
   - 选项：展示失败项供用户决策 / 手动修复 / 跳过本项

## light 专属

`light` 只在以下情况停下：

- 小需求目标不明确
- 无法定位到唯一文件或代码片段
- 修复会改变业务逻辑
- 最小自检失败且无法低风险修复

## standard / full 专属

1. **PRD 获取失败**
   - 选项：粘贴 PRD 原文 / 取消
2. **PRD 理解回执未经 ack**
   - 按 `ack-template.md` 输出回执
   - 选项：「全部正确，继续」(推荐) / 「需要修正」/ 「中止」
   - 未 ack 不得改代码
3. **代码现状探针未命中**
   - bug：强停，要求用户提供更精确定位、截图、操作路径，或改按 feat 处理
   - feat：可进入 `assumptions[]`，由理解回执 ack 兜底
4. **接口字段与现有 types 冲突**
   - 选项：扩展现有 type / 新建 type / 用户澄清
5. **命名歧义**
   - 选项：按 `rules/01-code-standards/naming.md` / 保留 PRD 原名 / 用户输入

## full 专属

1. **术语懒搜跨域未命中**
   - 选项：用户提供代码标识 / 标注为新术语并确认 AI 推断 / 中止
2. **ADR 草稿需要确认**
   - 选项：采纳并写入 / 编辑后写入 / 本次不留 ADR
