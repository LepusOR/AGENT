# 国际化（i18n）规范

> **技术栈**：`@alife/ads-common-utils` 的 `i18n`。
> **现状**：项目以国内商家为主，多语言优先级低；存量使用不规范，新代码请按本规范写。

---

## 🎯 5 条核心

1. 所有面向用户的文案走 `i18n.get({ id, dm })`（**保留扩展性**）；新代码直接中文也可接受，但**不要**用 `window.langMap` 这类混写
2. 参数传 **对象**，占位符 **语义化**（`{name}`，不是 `{0}`）
3. **禁止**手工 `.replace(/{xxx}/)`、**禁止**自制模板引擎（`{{x}}`）
4. `dm` 必填且必须是中文；避免在 `dm` 里塞复杂 HTML
5. `id` 用点号分层：`crmweb.{module}.{page}.{element}`

## 📦 初始化（入口一次）

```typescript
// src/common/i18n.ts
import enUS from '@alifd/next/lib/locale/en-us';
import zhCN from '@alifd/next/lib/locale/zh-cn';
import zhTw from '@alifd/next/lib/locale/zh-tw';
import { i18n } from '@alife/ads-common-utils';
import { syncLocale } from './cookie';

syncLocale();

let locale = zhCN;
if (i18n.lang === 'zh_TW') locale = zhTw;
else if (i18n.lang === 'en_US') locale = enUS;

let init = false;
if (!init) {
  i18n.init({
    namespaces: [
      'ads-bp',
      'ads-creative_sdk',
      'scp4p_ibp',
      'ads-ibp_ibp',
      'ads-mds-backup_ads-ibp',
      'ads-mds-backup_ads-bp',
    ],
    fallbackLanguage: 'zh_CN',
  });
  init = true;
}

export { locale };
export default i18n;
```

```tsx
// 入口
import { ConfigProvider } from '@alife/next';
import { locale } from '@/common/i18n';

<ConfigProvider locale={locale}>{children}</ConfigProvider>
```

| 字段 | 说明 |
|---|---|
| `i18n.lang` | 从 cookie 读，值为 `'zh_CN'` / `'zh_TW'` / `'en_US'` |
| `locale` | UI 组件库的语言包（给 `ConfigProvider`） |
| `namespaces` | 文案命名空间，**后写的覆盖前面** |
| `fallbackLanguage` | 回退语言，默认 `zh_CN` |

## 📝 `i18n.get` API

```typescript
interface I18nGetOptions {
  id: string;        // 唯一标识，层级命名
  dm: string;        // 默认文案（中文），找不到翻译时兜底
  replace?: boolean; // 是否翻译，默认 true；false 直接返回 dm
}

i18n.get(options, args?: Record<string, any> | any[]): string
```

## 🔑 带参数：必须对象传参

```typescript
// ✅ 对象传参，语义化占位符
i18n.get(
  { id: 'crmweb.welcome.message', dm: '欢迎 {name}' },
  { name: '张三' }
);
// → '欢迎 张三'

// ✅ 多占位符
i18n.get(
  { id: 'crmweb.star.result', dm: '您的平均{buyerLevel}商机浓度高于{density}%' },
  { buyerLevel: 'A类', density: 85 }
);

// ✅ 占位符可含字母 / 数字 / 下划线（正则 \w+）
i18n.get(
  { id: 'key', dm: '{user_name}在{year}年消费了{amount}元' },
  { user_name: '张三', year: 2024, amount: 1000 }
);

// ⚠️ 兼容旧版：数组传参 + 数字占位符（不推荐）
i18n.get({ id: 'key', dm: '{0}商机浓度高于{1}%' }, 'A类', 85);
```

### HTML 占位符（受控简单结构）

```tsx
// 仅"包一层简单标签 + 占位符" OK
<p
  dangerouslySetInnerHTML={{
    __html: i18n.get(
      { id: 'key', dm: '同行参与商家 <span style="color: #333">{num}</span>' },
      { num: item.supplierCount }
    ),
  }}
/>
```

## ❌ 三种不规范（必须废弃）

### 1. 手工 `.replace`

```typescript
// ❌
i18n.get({ id: 'key', dm: '{0}商机浓度' }).replace(/\{0\}/, guaranteedBuyerLevel);
i18n.get({ id: 'key', dm: '同行参与商家 {num}' }).replace(/{num}/g, item.supplierCount);

// ✅
i18n.get({ id: 'key', dm: '{buyerLevel}商机浓度' }, { buyerLevel: guaranteedBuyerLevel });
i18n.get({ id: 'key', dm: '同行参与商家 {num}' }, { num: item.supplierCount });
```

### 2. 自制模板引擎（`{{x}}`）

```typescript
// ❌ 自制 TemplateEngine 用 {{x}}
function TemplateEngine(tpl, data) { /* {{xxx}} 替换 */ }
TemplateEngine(i18n.get({ id, dm: '您有{{count}}张证书' }), { count: 5 });

// ✅ 用 i18n 内置对象传参（占位符是 {x}）
i18n.get({ id, dm: '您有{count}张证书' }, { count: 5 });
```

### 3. `dm` 里塞复杂 HTML

```tsx
// ❌ dm 含复杂嵌套 HTML 结构
<p dangerouslySetInnerHTML={{
  __html: i18n.get({ id, dm: `4、规则须知，<a href="..." target="_blank">点此查看</a>` }),
}} />

// ✅ 方案 A：拆成 JSX（推荐，无需多语言时）
<p>
  4、规则须知，<a href="..." target="_blank">点此查看</a>
</p>

// ✅ 方案 B：文案拆多条 i18n.get
<p>
  {i18n.get({ id: 'key1', dm: '4、规则须知，' })}
  <a href="..." target="_blank">{i18n.get({ id: 'key2', dm: '点此查看' })}</a>
</p>
```

## 🏷️ 命名规范

### `id`：`crmweb.{module}.{page}.{element}`

```
crmweb.authCenter.overview.title          // 认证首页标题
crmweb.authCenter.cert.upload.btn         // 证书上传按钮
crmweb.reportInspection.totalExportRevenue // 报告补验·年出口额
```

| 层 | 含义 | 示例 |
|---|---|---|
| 1 | 项目 | `crmweb` |
| 2 | 模块 | `authCenter` / `certRecommend` / `reportInspection` |
| 3 | 页面 / 功能 | `overview` / `cert` |
| 4 | 具体元素 | `title` / `btn` / `label` |

### `dm`

- **必填**、用**中文**
- 占位符 `{xxx}`（**不要** `{{xxx}}`）
- 尽量少 HTML；需要的话包一层 `<span>` 即可

### 新功能 / 重构策略

- **新功能**：优先 `i18n.get` 保留扩展；纯中文内联也可接受
- **重构**：不规范多语言（手工 replace / 自制模板）**直接改**成标准写法，改不动就先去掉多语言（留中文），等国际站统一再恢复

```typescript
// ✅ 推荐
const title = i18n.get({ id: 'auth.overview.title', dm: '认证首页' });

// ⚠️ 接受（简单文案、仅中文场景）
const title = '认证首页';

// ❌ 禁止混写
const title = window.langMap?.title || '认证首页';
```
